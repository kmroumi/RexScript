Why RexScript

Everything JS does. Done better.

RexScript is JavaScript under the hood. It compiles cleanly and runs anywhere JS runs. You keep your libraries, your ecosystem, your knowledge.

01 / Syntax

Concise by design

New keywords like fn, give, and loop replace verbose JavaScript constructs. Write 30% less boilerplate.

02 / Types

Optional, gradual typing

Add types when you need them. Skip them when you don't. Lighter than TypeScript, and built right in.

03 / Flow

Pipeline operator

Chain calls cleanly with ->. Data flows left to right, with no more nested parentheses.

04 / Safety

Null safety, first class

The exists keyword and ?/ null-coalesce operator make null checks readable and explicit.

05 / Ergonomics

Range literals

1..10 creates a range. Use it in loops, slices, and comparisons without importing a library.

06 / Compat

100% JS compatible

Compiles to clean, readable JS. Use any npm package. Call Rex from JS, or JS from Rex. Zero lock-in.

Origin

Why RexScript exists.

JavaScript is everywhere, but its syntax has sharp edges that trip up even experienced developers. Verbose function declarations, null checks that span entire lines, switch statements that fall through by default. These aren't bugs; they're design decisions that made sense decades ago. We write code today, not in 1995.

RexScript files those edges down. Not by replacing JavaScript, but by sitting cleanly on top of it. Every .rex file compiles to readable JavaScript that runs anywhere Node.js runs. Your libraries, your ecosystem, your knowledge: all intact.

Built by a team of students who love JavaScript but wanted something better. RexScript is the language we wished we had.

Syntax

Familiar, but evolved.

If you know JavaScript, you already know most of RexScript. The differences are small, deliberate, and always improve readability.

JavaScript
function greet(name) {
  return "Hello, " + name;
}

const users = ["Alice", "Bob"];
users.forEach(function(u) {
  console.log(u);
});
RexScript
fn greet(name: str): str {
  give `Hello, {name}`
}

let users = ["Alice", "Bob"]
users -> forEach(u => log(u))
JavaScript
for (let i = 0; i < 10; i++) {
  console.log(i);
}

const city = user
  && user.address
  && user.address.city
  || "Unknown";
RexScript
loop i in 0..9 {
  log(i)
}

let city = user?.address?.city
              ?/ "Unknown"
JavaScript
if (x !== null && x !== undefined) {
  process(x);
}

try {
  throw new Error("oops");
} catch (err) {
  console.error(err);
}
RexScript
if x exists {
  process(x)
}

try! {
  fail "oops"
} catch err {
  log.error(err)
}

Type System

Types, your way.

Fully optional. Annotate what matters, skip what doesn't. Types are inferred from context.

// Primitive types: str, num, bool, any, void
fn add(a: num, b: num): num {
  give a + b
}

// Type aliases
type UserId = str
type Point  = { x: num, y: num }

// Inferred. No annotation needed
let score = 42         // inferred: num
let name  = "Rex"     // inferred: str
let flag  = true      // inferred: bool

// Typed function with optional param
fn greet(name: str, loud?: bool): str {
  let msg = `Hello, {name}`
  give loud ? msg.toUpperCase() : msg
}

Static Analysis

What the compiler catches.

RexScript's static analyzer enforces these rules at compile time, before your code ever runs.

✓

Variables must be declared before use

✓

No redeclaration in the same scope

✓

give must appear inside a function body

✓

await must appear inside an async function

✓

Argument count must match declaration

✓

match wildcard _ must be the last arm

✓

guard else block must fail or give

✓

Type alias names share the variable scope — redeclaration is an error

✓

Loop variable is scoped to the loop body

✓

Typed parameters and return values on all functions

Get Started

Install RexScript

Up and running in under a minute. Requires Node.js 18+.

1

Clone the repo

Grab the source from GitHub.

git clone https://github.com/kmroumi/RexScript.git
2

Install deps

Requires Node.js 18+ and npm.

cd rexscript && npm install
3

Run a Rex file

RexScript compiles and executes in one step.

rex run app.rex
4

Compile to JS

Output clean, readable JS to deploy anywhere.

rex build app.rex --out dist/
